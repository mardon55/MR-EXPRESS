Mavzu: React + FastAPI + SQLite do'konida mahsulotlar soniga qo'yilgan barcha cheklovlarni (LIMIT) olib tashlash va to'liq ko'rsatish.Loyiha steki: FastAPI (Python), SQLite (SQLAlchemy ORM), React, Tailwind CSS (Glassmorphism).Muammo va Talab: Hozirda admin panel orqali qo'shilgan ba'zi mahsulotlar Mini Appda ko'rinmay qolyapti yoki dasturda cheklovlar bor. Admin yangi mahsulot qo'shganda HECH QANDAY LIMIT (cheklov) bo'lmasligi kerak. Admin xohlagancha (100 ta, 1000 ta va undan ko'p) mahsulot qo'shsa ham, ularning barchasi foydalanuvchilarga ko'rinishi shart.Vazifa: Loyihaning backend va frontend qismlarini quyidagi qat'iy qoidalar bo'yicha yangilab ber:1. Backend (FastAPI va SQLAlchemy) — LIMIT'larni olib tashlash:Mahsulotlarni bazadan tortib oluvchi barcha FastAPI endpointlarida (API routerlarida) SQL so'rovlaridan .limit() yoki .offset() funksiyalarini butunlay olib tashla.Masalan, quyidagi bo'limlar uchun so'rovlarni shunday yozki, bazadagi mos keluvchi barcha ma'lumotlarni cheklovsiz qaytarsin:Barcha mahsulotlar: db.query(Product).filter(Product.is_discount == False).all() (hech qanday .limit(20) qo'shilmasin).Chegirmalar bo'limi: db.query(Product).filter(Product.is_discount == True).all()Reels bo'limi: db.query(Product).filter(Product.is_reels == True).all()Kategoriyalar: db.query(Product).filter(Product.category_id == cat_id, Product.is_discount == False).all()2. Frontend (React) — Ma'lumotlarni to'liq qabul qilish:API'dan kelayotgan mahsulotlar massivini (array) frontendda qisqartirib ko'rsatadigan xavfsizlik mantiqlari (masalan, .slice(0, 10)) bo'lsa, ularni butunlay olib tashla.Kelgan barcha mahsulotlarni .map() yordamida Glassmorphism dizaynidagi kartochkalarda to'liq render qil. Ustma-ust yoki yonma-yon chiroyli joylashishi uchun Tailwind-ning grid grid-cols-2 md:grid-cols-3 gap-4 kabi moslanuvchan klasslaridan foydalan.Iltimos, admin qo'shgan har bir mahsulot Mini Appda foydalanuvchiga darhol va cheklovsiz ko'rinishini ta'minlovchi backend va frontend kodlarini to'liq va xatosiz yozib ber.
